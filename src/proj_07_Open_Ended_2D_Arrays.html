Project #07 - Open Ended 2D Array Project

Submission:

Submit on codepost.io.

Description:

Code something complex that requires you to make use of both 2D arrays and classes/objects in a substantial/meaningful manner. The game that you code needs to be at a level of complexity that will challenge you, but it also needs to be of a low enough level of complexity that you'll have time to finish it. This is why I'll be asking you to submit what you are planning to do in advance so that I can confirm your plan strikes the right balance in terms of overall complexity.

Requirements:

  1. The game needs to make use of 2D arrays in a substantial/meaningful manner. In other words, there should be a clear need to use a 2D array.
    1. Here are example of games that would be too basic to earn a decent score:
      • Tic-tac-toe
      • Checkers
      • Anything that's a knock-off of Connect 4.
      • Anything you could super easily write an AI to play a perfect game.
      • Pong
    2. Here are examples of games one person couldn't finish coding in their lifetime:
      • Battlefield
    3. Here are examples of board games you could make:
      • Battleship
      • Go
      • Chess
      • Reversi/Othello
      • Chinese Checkers
      • Stratego
    4. Here are video game examples of what you could make:
      • Minesweeper (???)
      • Snake (need to add items/powerups)
      • Tetris
      • Pac-man
      • 2048
      • A game you make up
      • An enhanced version of an existing game that adds more complexity
      • A simplified version of a tile-based combat game like Fire Emblem
      • A dungeon crawler
      • A very simplified version of a grid-based RPG (e.g., pokemon)
    5. Specific Coding Requirements
      1. The game must make use of a 2D array in a meaningful manner.
      2. The game must make heavy use of objects/classes in a meaningful manner.
      3. Your game needs to work. Code that errors out is an automatic 0.
      4. Your game needs to be fun to play.

    Tips:

    • Choose something to make that you really do want to make. If you are really interested in working on this, it will mostly feel fun to work on it rather than it feeling like homework.
    • This will be the most complex project you have worked on thus far. Be especially careful that you always have your code indented correctly. Test your code frequently or it will become nearly impossible to debug your code.
    • I'm happy to help. Please do be aware I'm not a super-human that can read all of the code you've written in 30 seconds and immediately find the bug you've been looking for. :)
