Project #05 - Calculator

Requirements:

Your calculator must have the following methods:
  1. addition two double parameters; return double result
  2. subtraction two double parameters (the result is the first parameter minus the second parameter); return double result
  3. multiplication two double parameters; return double result
  4. division two double parameters called num1 and num2; if num2 is 0, then immediately return the value 666.666-- otherwise, do the division as normal and return the double result
  5. square one double parameter; hint- use Math.pow(double base, double exponent) method; return double result
  6. isPerfectSquare one double parameter: return boolean value; hint: this method should check if a number if a perfect square... this info should be helpful...
    • Remember, perfect squares are numbers such as 1, 4, 9, 16, 25, 36, 49, 64, 81, 100, etc. This is because when you take the square root of these numbers, you get an integer (i.e., a number without decimals).
    • Use Math.sqrt(double num) to take the square root of a number.
    • If the double value num is an integer (i.e. has only 0's for a decimal), then the condition num % 1 == 0 will be true.
  7. squareRootWithTrick one double parameter called num; return double result; this method you write should do the following:
    • Call the method you just wrote to determine whether the number is a perfect square!!!
    • If num is not a perfect square, then take the square root of the number, and return the result.
    • If num is a perfect square, then return num. In other words, do not return the square root of num. Just return num without doing anything to it!! <--- This is the trick!!!

Some tips:

  • All I've asked you to do is write the methods. In other words, you don't need to build any sort of user interface to make the calculator usable. This isn’t a calculator that someone other than a programmer who knows how to call methods can use . You don't need a Scanner. You are not making a functioning calculator.
  • In your main method, you can call the calculator's methods as a way of testing them. However, when you turn your project in your main methods should be completely empty.
  • All the methods that you write should have public static in front of them like you've seen in class.

Reminder:

  • When you write these methods, they should be outside of the main method but still inside of the outermost pair of { }.
