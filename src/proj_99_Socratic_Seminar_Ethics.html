Description:

For a number of days in class, we will have a socratic seminar. These are the potential topics we will discuss as a class:
  • Socrates' Defense Speech (https://people.wku.edu/jan.garrett/103/Apol_Velasq_Tr.htm). It seems fitting for us to begin with for the first socratic seminar to start by discussing Socrates' famous speech that he gave just before his execution.

Requirements:

  1. In terms of specific ways in which you code this assignment, I'm looking for the below three main items. You will likely more than just one of each item below, but please make sure you implement each item at least once in your final product:
    • Create at least one class that extends another class.
    • Use super(parameter1, parameter2, etc.) at least once to call a constructor in the parent class.
    • Use super.insertMethodNameHere() to call a method in the parent class.
  2. Utilize inheritance in a meaningful manner.
  3. Similar to other assignments, I'm using the term "meaningful" as a way of giving you the freedom to exercise your creativity however you like in a manner that challenges you.
  4. A key difference between this assignment and previous assignments this school year is that in implementing inheritance, I'm not asking you to make use of inheritance to do something that you previously would have been unable to do without inheritance. As discussed in class, inheritance is an approach to writing code. So do not look for something to do that you wouldn't be able to do without inheritance. Instead, think of something that you already theoretically know how to do without inheritance but that you think would be easier to do with inheritance. This is incredibly subjective. So all I'm grading you on is if you choose to code what you think would be easier to do with inheritance. In other words, I'll take your word for it.
  5. As usual, code that errors out is an automatic 0. You'll need to revise it and show it to me again to receive credit.

Tips:

  • If you chose something you thought would be easier with inheritance but it turned out it wasn't, that is totally fine. You learned something valuable. It might be that you just prefer functional programming to object-oriented programming.
  • As usual, I'm happy to help you out as you are coding. Please continue to be aware I'm not a super-human that can read all of the code you've written in 30 seconds and immediately find the bug you've been looking for. :)
